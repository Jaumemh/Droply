# Sprint 6 - Tabla Si / Aleshores

## M1 - Comparticiones completas por sesion
- **Metrica:** `>= 0,6`
- **Si se cumple:** se mantiene el alcance actual y se valida que el flujo compartir -> abrir -> descargar funciona como ruta principal del MVP.
- **Si no se cumple:** se simplifica el flujo, se reducen pasos y se prioriza visibilidad del CTA principal de compartir.

## M2 - Subida de 10 MB
- **Metrica:** `<= 15 s`
- **Si se cumple:** la estrategia de subida por bloques y la barra de progreso quedan validadas como suficientes para Sprint 4.
- **Si no se cumple:** se prioriza optimizacion de red, se revisa tamanio de chunk y se retrasa cualquier mejora secundaria de UI.

## M3 - Interes en Pro
- **Metrica:** `>= 5%`
- **Si se cumple:** el modelo de extension de cuota mantiene sentido para una siguiente version.
- **Si no se cumple:** se replantea el valor del plan Pro y se potencia primero la propuesta gratuita.
